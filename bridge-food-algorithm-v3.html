Data Model (Complete)

Food Log Entry

{
  food_id: string (unique food identifier),
  kid_id: string (which child),
  timestamp: ISO-8601 (when this log was created),
  exposure_states: string[] (states achieved in THIS session),
    Valid values: ["looked_at", "touched", "smelled", "tasted", "ate"]
    Examples:
      ["looked_at"]  (kid just looked at banana)
      ["touched", "smelled"]  (parent guided through 2 steps)
      ["looked_at", "touched", "smelled", "tasted", "ate"]  (all in one session)
  parent_notes: string (optional notes from parent),
  
  // Computed fields (for query efficiency):
  highest_state_achieved: string (derived from all logs for this food)
    Values: "looked_at" (0.3x) | "touched" (0.5x) | "smelled" (0.6x) | 
            "tasted" (0.8x) | "ate" (1.0x)
  
  last_successful_state: string (for regression detection)
    Tracks the highest state achieved in the last 3 logs for this food
    If last_successful_state < highest_state_achieved, apply regression penalty
    Example: highest="ate", last_successful="looked_at" → kid regressed, apply penalty
}

State Multiplier Rationale

Why These Numbers? (Documented for Future Tuning)

State multipliers:

  • looked_at = 0.3x — Visual exposure only, lowest sensory engagement. Not meaningful confidence yet.
  • touched = 0.5x — Tactile contact, kid is braver. 67% jump from looking reflects significant behavioral step.
  • smelled = 0.6x — Olfactory engagement, high barrier for selective eaters. Only 20% increase from touch because olfactory alone (without tasting) is still preliminary.
  • tasted = 0.8x — Oral contact (may spit). This is the hard step in SOS. 33% jump from smelling because crossing into mouth is clinically significant.
  • ate = 1.0x — Swallowed, confirmed safe. 25% increase from tasting confirms acceptance.

Curve logic: Flatter at bottom (looking/touching closer together), steeper at top (tasted/ate farther apart) because SOS emphasizes oral desensitization as the hardest barrier.

IMPORTANT: These are starting values. Tune against real data after beta testing. If kids in beta show different barriers (e.g., smelling is scarier than tasting), adjust multipliers.

Weighted Baseline Calculation

Formula:

For each food in kid's history:
  1. Get highest_state_achieved (from all logs, ever)
  2. Get state_multiplier (0.3 to 1.0)
  3. Get most_recent_log_date (regardless of state)
  4. Calculate recency_weight (last 3 days = 1.0, drops to 0.1 at 30+ days)
  5. Combine: weight = state_multiplier × recency_weight
  6. IF kid has regressed: apply regression_penalty (e.g., 0.8x multiplier)

weighted_foods = [weight_1, weight_2, ..., weight_N]

baseline_profile = weighted_average(all_foods, weights)

Example: Calculate Baseline for Kid

Super Safe (always 0.60 base): [chicken nuggets, toast]
Regular Safe (always 0.40 base): [rice, cheese]
Explored (by state + recency):
  Apple: ate 3/20, most recent 3/20 → state=1.0 × recency=1.0 × no regression = 1.0
  Banana: looked 3/2, most recent 3/2 → state=0.3 × recency=0.1 = 0.03 (old)
  Seaweed: tasted 3/10, most recent 3/10 → state=0.8 × recency=0.5 = 0.4
  Sweet potato: ate 3/15, most recent 3/18 (looked again, regression!) 
    → highest=1.0, last_successful=0.3 → state=1.0 × recency=0.75 × regression=0.8 = 0.6

Final Baseline = weighted average of:
  nuggets (0.60), toast (0.60), rice (0.40), cheese (0.40),
  apple (1.0), banana (0.03), seaweed (0.4), sweet_potato (0.6)

Result: Baseline heavily weighted to recent ate/tasted foods, 
        with super-safe as persistent anchor (but scaled by exploration volume)

Super-Safe Scaling Cap (Claude Fix)

Problem (v2): Kid who ate nuggets at age 3 still anchors baseline 60% at age 7 as they explore 50+ foods.

Solution (v3): Scale super-safe contribution down as explored food list grows:

super_safe_weight = 0.60 / (1 + ln(num_explored_foods))

Examples:
  0 explored foods: super_safe = 0.60 (full weight)
  5 explored foods: super_safe = 0.60 / (1 + 1.6) = 0.23
  10 explored foods: super_safe = 0.60 / (1 + 2.3) = 0.18
  50 explored foods: super_safe = 0.60 / (1 + 3.9) = 0.11

This prevents super-safe foods from asymptotically anchoring everything.
As kid explores, super-safe recedes to "comfort zone" rather than "entire profile."

Trend Detection (Updated)

Deduplication Rule (Claude Fix)

Problem (v2): Parent logs "apple" 5x in a week. Trend detector sees "5/7 foods are sweet" (strong trend). Wrong.

Solution (v3): Deduplicate by food_id. Count distinct foods, not log events.

Last 7 log entries: [apple, apple, apple, banana, banana, carrot, seaweed]
Deduplicated to distinct foods (take highest state for each):
  Apple: ate (1.0x)
  Banana: looked (0.3x)
  Carrot: ate (1.0x)
  Seaweed: touched (0.5x)

Analyze on 4 distinct foods, not 7 log events

Minimum Sample Size (v3 Addition)

Rule: Don't activate trend detection unless kid has logged 5+ distinct foods.

Why: Fewer than 5 foods = noisy baseline. Super-safe/regular-safe should dominate, not trends.

If num_distinct_explored_foods < 5:
  Skip trend detection entirely
  Use baseline + variety pick only
  No trend-aligned bonus in distance calculation

Trend Strength Threshold

Count distinct foods matching a sensory dimension:

Threshold rules:
  6/7+ foods match → STRONG trend (apply full bonus)
  5/7 foods match → MODERATE trend (apply 50% bonus)
  4/7 or below → WEAK trend (don't apply bonus, use baseline)

Trend-aligned distance bonus:
  bonus = (trend_strength - 0.6) × 0.5
  
  So:
    5/7 (71%): bonus = (0.71 - 0.6) × 0.5 = 0.055 → 5.5% distance reduction
    6/7 (86%): bonus = (0.86 - 0.6) × 0.5 = 0.13 → 13% distance reduction
    7/7 (100%): bonus = (1.0 - 0.6) × 0.5 = 0.20 → 20% distance reduction

Recommendation Generation

Safe Pick (Distance < 2.5)

  • Closest to weighted baseline
  • Must be distinct from last 7 logged foods (no immediate repeats)
  • Explanation references recent foods: "Like the [recent food] you tried, but new"

Stretch Pick (Distance 2.5-4.5)

  • Moderate challenge, respects trend if present
  • Explanation: "You've been exploring [trend dimension]. Here's the next step."

Variety Pick (Nutritional Gap + Sensory Distance)

Algorithm:

  1. Identify missing food groups (0 vegetables, 0 proteins, etc.)
  2. Filter food database to foods in missing groups
  3. Find closest match by sensory distance (within 4.5 distance if possible)
  4. If closest is > 4.5 distance, use fallback (see below)
  5. Explanation: "You've explored mostly [current foods]. Let's add [group]."

Fallback (v3 Addition - Claude Fix): If no variety pick exists within 4.5 distance:

Option 1: Skip variety slot
  Return 2 safe/stretch picks instead
  Explanation to parent: "We're still building toward vegetables. When [child] 
  is ready, we'll recommend some great options."

Option 2: Recommend closest anyway
  Apply distance override (explain higher challenge)
  Use different explanation: "This is a bigger challenge, but you're making progress!"
  
Recommendation: Use Option 1 (don't force distance overrides on nutrition)

Edge Cases & Fallbacks

Edge Case Behavior Explanation to Parent
Safe pick doesn't exist (<2.5 distance) Return closest food regardless of distance + fallback explanation "New adventure, but we think you're ready"
Variety pick too far (>4.5 distance) Skip variety, return 2 safe/stretch picks "We're building toward vegetables. Coming soon!"
Fewer than 5 explored foods No trend detection, baseline only Simple recommendations, no trend language
Kid regressed (ate → looked_at) Apply 0.8x penalty to weight, don't de-recommend No explanation to parent yet (log it, tune v3.1)
Same food logged multiple times same week Deduplicate by food_id, take highest state Trend detection uses distinct foods, not log count
All recent logs are repeats of same food Trend detector sees 7/7 = same food, doesn't trigger Algorithm correctly identifies "kid stuck on one food"

Pseudo-Code (Final)

def generate_recommendations(kid_id: string) -> Recommendations:
    """Generate 3 daily recommendations using dynamic weighted baseline + exposure states."""
    
    # Load all food history
    all_logs = load_all_food_logs(kid_id)
    super_safe = load_super_safe(kid_id)
    regular_safe = load_regular_safe(kid_id)
    
    # Compute highest state + most recent date for each unique food
    food_profiles = {}
    for log in all_logs:
        if log.food_id not in food_profiles:
            food_profiles[log.food_id] = {
                highest_state: log.exposure_states[0],
                most_recent_date: log.timestamp,
                all_logs: [log]
            }
        else:
            # Update highest state and most recent date
            food_profiles[log.food_id].highest_state = max_state(
                food_profiles[log.food_id].highest_state,
                log.exposure_states
            )
            food_profiles[log.food_id].most_recent_date = max(
                food_profiles[log.food_id].most_recent_date,
                log.timestamp
            )
            food_profiles[log.food_id].all_logs.append(log)
    
    # Calculate weights for baseline
    weights = {}
    for food in super_safe:
        # Scale down as kid explores more
        scale = 0.60 / (1 + ln(len(food_profiles)))
        weights[food] = scale
    
    for food in regular_safe:
        scale = 0.40 / (1 + ln(len(food_profiles)))
        weights[food] = scale
    
    for food_id, profile in food_profiles.items():
        state_mult = STATE_MULTIPLIERS[profile.highest_state]
        days_ago = days_since(profile.most_recent_date, today())
        recency = calculate_recency_weight(days_ago)
        
        # Check for regression
        regression_penalty = 1.0
        last_3_logs = profile.all_logs[-3:]
        last_3_states = [log.exposure_states for log in last_3_logs]
        last_successful = max_state(last_3_states)
        if last_successful < profile.highest_state:
            regression_penalty = 0.8  # Apply penalty
        
        weights[food_id] = state_mult × recency × regression_penalty
    
    # Calculate baseline
    baseline = calculate_weighted_baseline(weights)
    
    # Detect trend (if 5+ explored foods)
    if len(food_profiles) >= 5:
        last_7_unique = deduplicate_last_7_logs(all_logs)
        trend = detect_trend(last_7_unique)
        trend_strength = max(trend.values())
    else:
        trend = None
        trend_strength = 0
    
    # Generate recommendations
    all_foods = load_food_database()
    distances = {}
    for food in all_foods:
        if is_blacklisted(food, kid_id):
            distances[food] = infinity
        else:
            base_dist = calculate_distance(food, baseline)
            
            # Apply trend bonus if strong trend
            if trend_strength > 0.6 and food_matches_trend(food, trend):
                bonus = (trend_strength - 0.6) × 0.5
                base_dist = base_dist × (1 - bonus)
            
            distances[food] = base_dist
    
    # Safe pick
    safe_pick = find_closest_food(
        distances,
        max_distance=2.5,
        exclude=last_7_unique
    )
    if not safe_pick:
        safe_pick = find_closest_food(distances, exclude=last_7_unique)
        safe_explanation = "New adventure, but we think you're ready"
    else:
        safe_explanation = explain_safe(safe_pick, last_7_unique, trend)
    
    # Stretch pick
    stretch_pick = find_closest_food(
        distances,
        min_distance=2.5,
        max_distance=4.5,
        exclude=[safe_pick],
        prefer_trend_aligned=True
    )
    stretch_explanation = explain_stretch(stretch_pick, trend)
    
    # Variety pick
    missing_groups = find_missing_food_groups(food_profiles)
    variety_pick = None
    variety_explanation = None
    
    if missing_groups:
        closest_in_gap = find_closest_food(
            distances,
            food_group=missing_groups[0],
            max_distance=4.5,
            exclude=[safe_pick, stretch_pick]
        )
        
        if closest_in_gap and distances[closest_in_gap] <= 4.5:
            variety_pick = closest_in_gap
            variety_explanation = explain_variety(variety_pick, trend, missing_groups[0])
    
    # Fallback if no variety pick
    if not variety_pick:
        # Skip variety slot, return only 2 picks
        variety_pick = None
        variety_explanation = "We're building toward {food_group}. Coming soon!"
    
    return [
        Recommendation("Safe Pick", safe_pick, safe_explanation),
        Recommendation("Stretch Pick", stretch_pick, stretch_explanation),
        Recommendation("Variety Pick", variety_pick, variety_explanation) if variety_pick else None
    ].filter(not None)
