🧠 Bridge Food Recommendation Algorithm

Purpose: Generate safe, progressive food recommendations for children with selective eating based on multi-dimensional sensory analysis.

Why Rules-Based (Not LLM)

  • Transparency: Parents see exactly WHY we recommend each food (e.g., "same texture, similar flavor, warm temperature")
  • Cost: $0/month vs. $2,150/year for LLM. Phase 1 focus: prove product-market fit, not premium features
  • App Store Safety: Health apps must have explainability. Rules = defensible. LLM = "why did it suggest sushi to a kid with 4 foods?"
  • Control: Zero hallucinations. You own every recommendation. No surprises.

Sensory Profile Model (6 Dimensions)

Every food is mapped across these sensory axes:

Texture (0-10 scale)

  • 0: Pure liquid (juice, milk)
  • 2: Soft/mushy (applesauce, oatmeal)
  • 4: Soft/chewy (bread, pasta)
  • 6: Mixed (chicken nuggets: crunchy exterior, soft interior)
  • 8: Crunchy (chips, apples, carrots)
  • 10: Hard/brittle (raw carrot, nuts)

Flavor (5 sub-dimensions)

  • Sweet: 0-10 (none to very sweet)
  • Salty: 0-10 (none to very salty)
  • Savory: 0-10 (umami/meaty)
  • Sour: 0-10 (acidic)
  • Bitter: 0-10 (astringent)

Temperature

  • Cold: 0-5°C (ice cream, cold juice)
  • Cool: 5-15°C (yogurt, cheese)
  • Room temp: 15-25°C (bread, cookies)
  • Warm: 25-45°C (warm applesauce)
  • Hot: 45°C+ (soup, oatmeal)

Color

  • Red, Orange, Yellow, Green, Brown, White, Mixed/Multi-color
  • Also track: color intensity (pale vs. bright)

Mouthfeel

  • Juicy, Dry, Creamy, Crispy, Tender, Chewy

Preparation Method

  • Raw, Boiled, Baked, Fried, Steamed

Baseline Calculation

Input (from Onboarding):

  • Safe foods: 1-5 foods kid eats regularly
  • Super safe foods: 2-3 foods kid is obsessed with (subset of safe)

Weighting:

Super Safe weight: 60%
Regular Safe weight: 40%

Profile_avg = (0.60 × avg(super_safe)) + (0.40 × avg(regular_safe))

Example: Kid eats [chicken nuggets (super safe), toast (super safe), rice (regular), cheese (regular)]

Baseline Profile will be:
- 60% weighted toward: crunchy, salty, warm, brown (nuggets + toast)
- 40% weighted toward: soft, bland, warm, white (rice + cheese)

Result: Kid is comfort with WARM, SLIGHTLY-SALTY, MIXED-TEXTURE, BROWN foods

Distance Calculation

For each potential bridge food, calculate multi-dimensional distance:

Distance² = w_texture × (texture_diff)² 
          + w_flavor_salty × (salty_diff)² 
          + w_flavor_savory × (savory_diff)²
          + w_temp × (temp_diff)²
          + (... other dimensions)

Distance = √(Distance²)

Default Weights:

Dimension Weight Why
Texture 0.15 Most critical sensory factor
Flavor (Salty) 0.12 Kids prefer familiar flavors
Temperature 0.10 Kids have temperature preferences
Color 0.08 Visual appeal matters
Bitter 0.08 Bitter is a big turn-off for kids

Distance Interpretation:

  • 0-2.5: "Super Close" - Kid can try this today
  • 2.5-4.5: "Stretch Pick" - Challenging but doable
  • 4.5-6.5: "Difficult" - Maybe week 3+
  • 6.5+: "Too Far" - Don't recommend

Daily Recommendation Strategy

Each day: Generate 3 recommendations

Safe Pick (Distance < 2.5)

  • Highest similarity to baseline
  • Low risk, high success chance
  • Role: "This is close to what you like"
  • Example: Kid likes crunchy salty chicken nuggets → try crispy seaweed snacks

Stretch Pick (Distance 2.5-4.5)

  • Moderate challenge
  • One sensory dimension different (e.g., softer but same flavor)
  • Role: "Try this tiny challenge"
  • Example: Kid likes crunchy → try soft but similar flavor profile

Variety Pick (Nutritional Gap Targeting)

  • Identifies if kid's diet is unbalanced (e.g., only warm foods, only brown/white colors)
  • Recommends different sensory category but still within 4.5 distance if possible
  • Role: "Let's add more colors/textures"
  • Example: Kid eats brown/white foods only → recommend yellow (sweet potato, banana) or orange (carrots)

Novelty Control

Last 7 days:  Don't recommend same food twice
Last 30 days: Lower weight by 50% if recommended before
Last 90 days: Available again at full weight

Prevents boredom and encourages true exploration.

How It Handles Infinite Foods

  • Dimensionality: Kid with only 4 safe foods → algorithm calculates sensory baseline from those 4 → ranks all 100+ foods by similarity
  • Progressive Steps: Algorithm never jumps sensory distance. If kid likes warm salty crunchy, it recommends warm salty soft BEFORE jumping to cool bitter vegetables
  • Diversity Weighting: If kid has eaten crunchy foods 5x this week, algorithm weights soft/mixed textures higher next recommendations
  • Personalization: Each kid gets a unique recommendation order based on THEIR baseline, not generic "picky eaters eat these foods"

Example Recommendation Flow

Day 1 - Safe Pick: Seaweed snacks (Distance: 1.8)
"We picked this because: Crunchy like chicken nuggets, salty like you like it, but something new to try!"

Day 1 - Stretch Pick: Soft sweet potato fries (Distance: 3.2)
"Still warm like you like it, a little softer than usual, but pretty similar. Maybe try just one bite?"

Day 1 - Variety Pick: Yellow bell pepper slices
"Your week has been brown & white foods. Let's add some color! This is sweet & crunchy like things you like."

Day 2 - Safe Pick: Crispy bacon (Distance: 1.5)
"Crunchy, salty, warm - all your favorite things with a fun new flavor!"

Week 2 - Shift Detected
Kid logged: mostly crunchy foods (7 of last 10 foods)
→ Algorithm increases soft texture weight by 30% for next week's recommendations
→ Still respects baseline (won't jump to something that's 90% different)

Implementation & Tuning

  • Phase 1: Build algorithm with default weights, test with beta families (week 1-2)
  • Phase 1 Results: Track which recommendations kids actually try (Safe vs. Stretch vs. Variety success rates)
  • Phase 1 Tuning: Adjust weights based on real data (e.g., if Stretch picks fail 80% of the time, widen distance gap)
  • Phase 2: If working well, lock in weights. Consider A/B testing variants later.
  • Never: Switch to LLM unless rules-based stops working (unlikely)

Key Decisions Made

  • Super Safe Weighting (60/40): Starting point. Tune after beta feedback. If kids make jumps too fast, increase to 70%. If stuck, lower to 50%.
  • Daily Strategy (3 picks): Balanced between safety and challenge. If engagement is low, could increase to 5 picks. Monitor beta feedback.
  • Distance Weights: Texture (0.15) is highest because it's the most sensitive. Tune individually per sensory dimension based on beta data.
