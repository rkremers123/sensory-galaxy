How It Works (Overview)

Every time the app generates recommendations:

  1. Load kid's entire food log (all foods ever explored)
  2. Weight each food by safety level + recency
  3. Calculate sensory baseline from weighted foods
  4. Detect trend in last 7 foods logged
  5. Generate 3 recommendations that respect baseline + trend
  6. Explain why (reference recent foods + detected trend)

Multi-Weighted Baseline

Each food in the kid's history gets a composite weight:

Weight Formula

weight = safety_level × recency_weight

safety_level:
  - Super Safe (nuggets, toast): 0.60 (always highest)
  - Regular Safe (rice, cheese): 0.40 (always high)
  - Explored (logged, ever): varies by recency (see below)

recency_weight:
  - Logged last 3 days: 1.00 (100%)
  - Logged 4-7 days ago: 0.75 (75%)
  - Logged 8-14 days ago: 0.50 (50%)
  - Logged 15-30 days ago: 0.25 (25%)
  - Logged 30+ days ago: 0.10 (10%)

Example: Calculate Baseline for Kid

Kid's Food History:

Super Safe (always 0.60):
  - Chicken Nuggets
  - Toast

Regular Safe (always 0.40):
  - Rice
  - Cheese

Explored (weighted by recency):
  - Seaweed (logged 2 days ago): weight = 1.00 (very recent, high confidence)
  - Sweet Potato Fries (logged 5 days ago): weight = 0.75
  - Shrimp (logged 10 days ago): weight = 0.50
  - Broccoli (logged 25 days ago): weight = 0.25
  - Carrots (logged 45 days ago): weight = 0.10

Baseline Calculation:

For each sensory dimension (texture, flavor, temperature, etc.):

Baseline = (nuggets × 0.60) + (toast × 0.60) + (rice × 0.40) + (cheese × 0.40)
         + (seaweed × 1.00) + (sweet_potato × 0.75) + (shrimp × 0.50) 
         + (broccoli × 0.25) + (carrots × 0.10)

Result: A sensory profile that heavily emphasizes:
  - What they always eat (super safe)
  - What they recently tried (recency boost)
  - What they tried long ago (low weight, fading influence)

Key insight: Baseline shifts every day as kid logs new foods.

Recency Weighting Details

Why recency matters:

  • Kid tried carrots once 2 months ago = low confidence they'd eat it again
  • Kid tried seaweed 3 days ago = high confidence in that sensory profile
  • Recent exploration shows what kid is currently open to

Practical example:

Week 1: Kid tries [apple, banana, sweet potato, honey crackers]
  Baseline at end of Week 1: Strong sweet flavor, room temp bias

Week 2: Kid tries [broccoli, spinach, green beans] (parent nudge)
  Baseline Week 2: Now includes 50% bitter/veggie, but still 60-70% weighted to Week 1
  → Recommendations won't jump to all vegetables (respects recent sweet exploration)

Week 3: Kid tries [mango, peach, vanilla pudding, yogurt]
  Baseline Week 3: Very strong sweet + creamy profile now
  → Recommendations lean into sweet + creamy variants

Trend Detection (Last 7 Foods)

Detect what sensory dimensions the kid is gravitating toward:

Trend Analysis

Last 7 foods logged: [seaweed, sweet potato fries, apple, banana, mango, yogurt, crackers]

Analyze each dimension:
  - Texture: crunchy (5/7), soft (2/7) → TREND: CRUNCHY (71%)
  - Flavor: sweet (6/7), salty (2/7), savory (1/7) → TREND: SWEET (86%)
  - Temperature: room temp (4/7), warm (3/7) → WEAK (57% room temp, not strong)
  - Color: natural colors (7/7) → NOT DISTINCTIVE
  
Trend Strength Threshold:
  - 6/7 or higher = STRONG trend
  - 5/7 = MODERATE trend
  - 4/7 or lower = WEAK (don't override baseline)

In this case:
  STRONG trend: Sweet + Crunchy
  → Recommendations should honor this (lean into sweet crunchy options)
  → But still respect baseline (don't ignore safe foods)

Distance Calculation

Same 6-dimensional model as v1, but applied to dynamic baseline:

Distance² = w_texture × (texture_diff)² 
          + w_flavor_sweet × (sweet_diff)²
          + w_flavor_salty × (salty_diff)²
          + w_flavor_savory × (savory_diff)²
          + w_flavor_sour × (sour_diff)²
          + w_flavor_bitter × (bitter_diff)²
          + w_temp × (temp_diff)²
          + w_color × (color_diff)²
          + w_mouthfeel × (mouthfeel_diff)²
          + w_prep × (prep_diff)²

Distance = √(Distance²)

Distance Interpretation (same as v1):
  0-2.5: Super Close
  2.5-4.5: Stretch Pick
  4.5-6.5: Difficult
  6.5+: Too Far

Daily Recommendation Strategy

Safe Pick (Distance < 2.5)

  • Closest to current weighted baseline
  • Low risk, high success
  • Should align with trend if possible
  • Explanation: Reference recent foods ("like seaweed you tried last week")

Stretch Pick (Distance 2.5-4.5)

  • Moderate challenge
  • Respects trend (if sweet trend, don't suggest bitter vegetables)
  • Explanation: "You've been exploring sweet foods. Try raisins (sweet + chewy, new texture)"

Variety Pick (Nutritional Gap)

  • Identifies what kid hasn't explored (vegetables, proteins, grains)
  • If strong trend present, tries to slide variety into trend
  • Example: Kid trending sweet → suggest sweet potato (sweet + vegetable)
  • Explanation: "You've been exploring sweet foods. Let's add a vegetable. Sweet potato is sweet AND a vegetable."

Pseudo-Code Implementation

def generate_recommendations(kid_id: string) -> List[Recommendation]:
    """Generate 3 daily recommendations using dynamic weighted baseline."""
    
    # Load all food history
    super_safe = load_super_safe(kid_id)
    regular_safe = load_regular_safe(kid_id)
    all_explored = load_all_explored_foods(kid_id)
    
    # Calculate weights for each food
    weights = {}
    for food in super_safe:
        weights[food] = 0.60  # Always 60%
    for food in regular_safe:
        weights[food] = 0.40  # Always 40%
    for food in all_explored:
        days_ago = days_since_logged(food, kid_id)
        recency = calculate_recency_weight(days_ago)
        weights[food] = recency  # Varies by recency
    
    # Calculate dynamic baseline
    baseline_profile = calculate_weighted_baseline(
        super_safe, regular_safe, all_explored, weights
    )
    
    # Detect trend in last 7 logs
    recent_7 = get_last_7_foods_logged(kid_id)
    trend = detect_trend(recent_7)  # Returns {dimension: strength}
    trend_strength = max(trend.values())
    
    # Calculate distances for all foods
    all_foods = load_food_database()
    distances = {}
    for food in all_foods:
        if is_blacklisted(food, kid_id):
            distances[food] = infinity
        else:
            base_distance = calculate_distance(food, baseline_profile)
            # Adjust distance if food aligns with trend
            if trend_strength > 0.7 and food_matches_trend(food, trend):
                base_distance *= 0.8  # 20% bonus for trend-aligned
            distances[food] = base_distance
    
    # Generate 3 recommendations
    safe_pick = find_closest_food(distances, max_distance=2.5, recent_7)
    stretch_pick = find_closest_food(distances, 2.5, 4.5, recent_7, trend)
    variety_pick = find_nutritional_gap(distances, baseline, trend, recent_7)
    
    # Generate explanations
    return [
        Recommendation(safe_pick, "Safe Pick", explain_safe(safe_pick, recent_7)),
        Recommendation(stretch_pick, "Stretch Pick", explain_stretch(stretch_pick, trend)),
        Recommendation(variety_pick, "Variety Pick", explain_variety(variety_pick, trend, nutritional_gap))
    ]

def calculate_recency_weight(days_ago: int) -> float:
    """Return weight based on how long ago food was logged."""
    if days_ago <= 3:
        return 1.00
    elif days_ago <= 7:
        return 0.75
    elif days_ago <= 14:
        return 0.50
    elif days_ago <= 30:
        return 0.25
    else:
        return 0.10

def explain_safe(food, recent_7):
    """Generate explanation for Safe Pick."""
    similar_recent = find_similar_foods(food, recent_7)
    if similar_recent:
        return f"Like the {similar_recent[0].name} you tried recently, but with a fun new flavor!"
    else:
        return f"Very similar to your favorite foods. You'll probably love this."

def explain_stretch(food, trend):
    """Generate explanation for Stretch Pick."""
    if trend:
        return f"You've been exploring {trend['dominant_flavor']} foods. {food.name} is similar, but with a new texture."
    else:
        return "A small challenge, but based on what you've been trying, we think you're ready."

def explain_variety(food, trend, gap):
    """Generate explanation for Variety Pick."""
    if trend and food_bridges_trend_to_gap(food, trend, gap):
        return f"You love {trend['dominant_flavor']} foods. {food.name} is {trend['dominant_flavor']} AND includes {gap} (which you haven't explored much)."
    else:
        return f"You've explored mostly {trend['dominant']} foods. Let's add some {gap} variety."
