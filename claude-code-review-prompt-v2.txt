### Context
I'm building Sensory Galaxy, an iOS app for kids with selective eating using the Sequential Oral Sensory (SOS) method.

The core feature is a "bridge food recommendation engine" that:
- Collects a child's food exploration history (every food they log)
- Uses dynamic weighting (recency + safety level) to calculate a constantly-updating sensory baseline
- Detects trends in recent exploration (what are they gravitating toward?)
- Generates 3 daily recommendations that respect baseline + momentum

### Key Design Decisions in v2
1. **Dynamic baseline:** Recalculates every time a new food is logged (not static like v1)
2. **Recency weighting:** Foods logged 3 days ago count 10x more than foods logged 30+ days ago
3. **Trend detection:** Analyze last 7 foods to detect "this kid is trending toward sweet + crunchy"
4. **Trend-aligned recommendations:** If kid is trending sweet, don't recommend bitter vegetables. Honor their momentum.
5. **Explanations:** Every recommendation explains why ("You've been exploring sweet foods, so we suggest...") + references recent foods
